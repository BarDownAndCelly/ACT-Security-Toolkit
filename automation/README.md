# Automation

Automation is the next step in applying AI to cybersecurity.

While prompt engineering helps security professionals interact effectively with Large Language Models (LLMs), automation enables those capabilities to be integrated into repeatable workflows, security operations, and investigative processes.

This section of the repository focuses on practical automation examples that improve efficiency without replacing human expertise.

---

## Objectives

The automation section aims to:

* Demonstrate production-quality cybersecurity scripting
* Reduce repetitive investigative and administrative tasks
* Showcase AI-assisted security workflows
* Integrate Microsoft security platforms with modern AI capabilities
* Build reusable components for future AI agents

---

## Repository Structure

```text
automation/
├── powershell/
├── python/
```

---

## PowerShell

PowerShell examples focus on Microsoft security technologies, including Microsoft Graph, Microsoft Purview, Microsoft Defender, Entra ID, Exchange Online, and Microsoft 365 administration.

---

## Python

Python examples demonstrate investigation automation, data processing, API integrations, AI-assisted analysis, and security tooling that complements enterprise security operations.

---

## Philosophy

Automation should remove repetitive work, improve consistency, and accelerate investigations while keeping humans responsible for security decisions, evidence validation, and organizational risk management.
