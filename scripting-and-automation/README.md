# Scripting and Automation

Scripting and automation allow security professionals to spend less time performing repetitive administrative tasks and more time investigating, engineering, and improving security outcomes.

Within the ACT Security Toolkit, scripting and automation support Insider Threat operations through practical examples using Microsoft technologies, PowerShell, Python, and AI.

Just like AI, Automation should never replace engineering.

It should amplify it.

---

# Objectives

The scripting and automation section aims to:

- Demonstrate production-quality security engineering
- Automate repetitive administrative and investigative tasks
- Build reusable Microsoft security tooling
- Support Insider Threat operations through automation
- Integrate Microsoft security technologies with AI
- Build reusable components for future AI agents and automation workflows

# Philosophy

Automation should remove repetitive work—not critical thinking.

Security engineers should understand every script they run, every API they call, and every action performed within their environment.

The goal of automation is not to replace people.

The goal is to eliminate repetitive work so engineers and investigators can spend more time solving problems that require experience, context, and human judgment.

AI is another tool available to security professionals.

Like every major technological advancement before it, AI creates new opportunities for defenders and new opportunities for adversaries.

The responsibility of security professionals is to understand the technology, govern it responsibly, and use it to strengthen—not replace—the human element of Insider Threat investigations.
