# ACT Documentation

This directory contains documentation about the ACT Security Toolkit itself.

Unlike the training, scripting, workflows, and quick reference sections, the documents contained here focus on how the project is organized, maintained, and developed.

If you're looking for Insider Threat guidance, Microsoft security information, or automation examples, you'll find those elsewhere in the repository.

---

# Purpose

The documentation in this directory exists to ensure the ACT Security Toolkit remains consistent, maintainable, and easy to contribute to as it continues to grow.

Topics include:

- Repository architecture
- Engineering standards
- Development roadmap
- Contribution guidelines
- Project history and release notes

# Philosophy

Good documentation is just as important as good code.

Clear documentation makes engineering decisions understandable, reduces onboarding time, encourages consistent contributions, and helps ensure the toolkit remains useful long after the original code was written.

The goal of this directory is to document **how the ACT Security Toolkit is built**, allowing the rest of the repository to focus on helping security professionals build and mature Insider Threat programs.
